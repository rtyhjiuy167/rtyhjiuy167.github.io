3 接口文档

发表于 2021-08-15 分类于 API

实战开发中后端会开发很多接口

每个接口都有大量的参数

思考：前端如果想知道有哪些接口并且每个接口有几个参数？

回答：通过接口文档

思考：接口文档自己写还是怎样？

回答：有很多方法，可通过apiDoc来生成

apiDoc是什么？

就是nodejs中的一个模块

去哪下

https://apidocjs.com/

如何用

步骤1：先下载模块，后期通过命令基于于注释生成文档（下载到全局，一次即可）

步骤2：在项目根目录创建 apidoc.json 文件

步骤3：去写接口注释

步骤4：生成接口文档

apidoc -i 接口注释目录 -o 接口文档存放目录

步骤5：查看doc文件下的index.html

3 基本命令

发表于 2021-08-15 分类于 mongodb

db查看当前操作的数据库

show dbs或show databases显示所有数据库

use 数据库名称切换到指定的数据库（如果没有，则会隐式创建）

db.createCollection('集合名（数据库名）')创建此时操作的数据库下的一个集合

db.集合名.drop()删除此时操作的数据库下的一个集合

db.dropDatabase()切换到要删除的数据库，执行该命令即可删除该数据库

db.集合.insertOne(key:value)插入数据（集合没有，则隐式创建）

show collections显示当前数据库的全部集合

db.集合.find()查看集合的所有数据

构造树对象

我们可以通过update-index、write-tree、read-tree等命令来构建树对象并塞入到暂存区

提交对象

执行$ git cat-file -p 74245b5b24ca3a29bb6a534d62c38600f6664056，查看返回的哈希值的内容：

一个树对象(tree)本质上就是一个项目的版本，而提交对象(commit)指示对树对象进行了封装，提交对象是链式的（虽然提交对象之间是链式的，但只有知道某个提交对象的哈希值，就可直接用哈希值回退到该的提交对象），且存储了注释与作者等
