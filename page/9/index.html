6 存储

发表于 2021-08-15 更新于 2021-08-14 分类于 git

git stash：该命令会将未完成的修改保存到一个栈上
git stash list：查看存储
git stash apply：应用栈顶的存储（但不会删除）
git stash apply stash@{索引值} ：应用指定的存储（但不会删除）
git stash pop stash@{索引值} ：应用指定的存储并删除
git stash drop stash@{索引值}：删除存储

实际上每次存储，git都会做一次提交，在git log --oneline --decorate --graph --all显示完整的项目分叉历史下仍能看见，但在git log --oneline是看不见的

6-2 npm与packagejson

发表于 2021-08-15 分类于 nodejs

package.json

每一个项目都应有一个 package.json 文件，通过npm init创建
之后写入项目名，name(默认为npm-demo)
版本号，version(默认1.0.0)
项目的描述信息 description
项目入口 entry point(默认为index.js)
text command
github仓库地址 git repository
关键字 keywords
作者 author
开源许可证 license(默认为ISC)

在创建完package.json 文件之后，下载第三方包时，应加上--save，这样可以在package.json 文件中看见依赖信息dependencies

如果node_modules丢失，可在项目根目录终端中输入npm install。其在有package.json 文件时会找到该文件里的dependencies，将其中的所有依赖项都下载一遍

npm

nmpnode package manager

nmp是在下载安装完node就有了的软件

命令

nmp init

​ -nmp init -y可以跳过向导，快速生成

``nmp install`
一次性把dependencies选项中的依赖项全部安装

nmp install 包名
-只下载包
-简写：npm i 包名

nmp install --save 包名或nmp install 包名 --save
-下载包并保存依赖项
-简写：npm i -S 包名

npm uninstall 包名
只删除包，保留依赖项
-简写：npm un 包名

npm uninstall --save 包名
删除包与依赖项
-简写：npm un -S 包名

npm help
-查看使用帮助

npm 命令 --help
查看指定命令的使用帮助
例如npm install --help

npm被墙问题

安装淘宝的cnpmnpm install --global cnpm
之后把要原本要安装包时的命令中的npm换成cnpm
这样就从国外的npm服务器换成的淘宝的服务器了

如果不想下载，可每次安装包时可npm install 包名 --registry=https://registry.npm.taobao.org
但是每次手动这样加参数会很麻烦，我们可以把这个加入配置文件中：npm config set registry https://registry.npm.taobao.org
查看npm配置信息npm config list
